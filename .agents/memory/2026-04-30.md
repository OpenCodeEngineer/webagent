# 2026-04-30

## OpenClaw Gateway SIGTERM Crash Loop — FIXED

### Root Cause
The SIGTERM handler was in `dist/gateway-cli-DkZMw8r-.js` line 422 (not `dist/index.js`). The function `onSigterm()` called `request("stop", "SIGTERM")` which triggered a full graceful shutdown. An unknown root process sends SIGTERMs every ~20 seconds, causing a crash loop (restart counter hit 1409).

### Fix Applied (VM hot-patch)
Patched `/home/openclaw/openclaw/dist/gateway-cli-DkZMw8r-.js` line 422:
```js
// Before:
const onSigterm = () => {
    gatewayLog$1.info("signal SIGTERM received");
    request("stop", "SIGTERM");
};

// After:
const onSigterm = () => { gatewayLog$1.info("SIGTERM blocked by patch"); return;
    gatewayLog$1.info("signal SIGTERM received");
    request("stop", "SIGTERM");
};
```

Combined with existing `ignore-sigterm.js` preload (catches signals before app handler) and `KillSignal=SIGKILL` in systemd unit (so systemctl stop still works).

### Result
- Gateway stable with 0 restarts since fix
- SIGTERMs still arriving from unknown root process but harmlessly blocked
- Health endpoints all responding: `/health` 200, `/health/openclaw` {"status":"ok"}, `/widget.js` 200
